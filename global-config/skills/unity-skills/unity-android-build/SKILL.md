# Unity Android Studio Export & Gradle Builds

## Overview
Guidelines for managing Unity Android Studio exports, Gradle builds, automated upload commands, and setting up native Android debug symbols for crash telemetry.

## When to Use
- Exporting the Unity project to an Android Studio project structure.
- Troubleshooting build shell scripts (`build_upload.command`).
- Fixing directory issues where Gradle commands fail due to missing `./gradlew` files in the root export path.
- Configuring upload procedures for `.symbols.zip` files to Google Play Console or Firebase Crashlytics.

## Key Concepts

| Concept | Description |
|---------|-------------|
| **Export Project** | Unity Build Settings option to output raw Gradle/Android sources instead of an APK/AAB directly. |
| **Gradle Wrapper (gradlew)** | Script that manages execution of Gradle tasks. Must be placed in the root directory of the exported source. |
| **build_upload.command** | Custom shell script that compiles the AAB and automatically uploads to distribution channels. |
| **Debug Symbols** | Native symbols (symbols.zip) generated by Unity to demangle C++ call stacks from IL2CPP crashes. |

## Best Practices
- ✅ **Always** verify that the working directory of your gradle scripts matches the directory containing `./gradlew`.
- ✅ Use absolute paths or relative shell wrappers (`$(dirname "$0")`) inside build command files.
- ✅ Automate the zipping and uploading of `symbols.zip` files as part of the post-build gradle target.
- ✅ Store keystore credentials in environment variables or `local.properties` rather than hardcoding them in VCS tracked files.
- ❌ **NEVER** attempt to run `./gradlew` commands inside a path that has spaces unless paths are properly escaped in the shell.
- ❌ **NEVER** ignore gradle build cache issues; run `./gradlew clean` if changes to resources or manifest do not apply.

## Few-Shot Examples

### Example 1: Resolve gradlew directory mismatch in build_upload.command
**User**: "Sửa lỗi script build_upload.command báo Không thấy ./gradlew khi chạy build xuất từ Unity."

**Agent**:
```bash
#!/bin/bash
# Move to the script's directory first to prevent context mismatches
cd "$(dirname "$0")"

echo "Checking build directory..."
if [ ! -f "./gradlew" ]; then
    echo "Error: ./gradlew not found. Please place this script in the root of the exported Android Studio project."
    exit 1
fi

echo "Cleaning build cache..."
chmod +x ./gradlew
./gradlew clean

echo "Building Release Bundle (AAB)..."
./gradlew :launcher:bundleRelease

if [ $? -eq 0 ]; then
    echo "Build Succeeded!"
    # Call upload script or API here
else
    echo "Build Failed!"
    exit 2
fi
```

### Example 2: Automatic Upload of Debug Symbols
**User**: "Làm sao để cấu hình tự động upload debug symbols lên Firebase Crashlytics sau khi build gradle."

**Agent**:
```groovy
// In the launcher/build.gradle file:
apply plugin: 'com.google.firebase.crashlytics'

android {
    buildTypes {
        release {
            minifyEnabled true
            // Enable crashlytics custom mapping upload
            firebaseCrashlytics {
                nativeSymbolUploadEnabled true
                strippedSymbolsDir "$projectDir/src/main/jniLibs/arm64-v8a"
                unstrippedSymbolsDir "$projectDir/src/main/jniLibs/unstripped/arm64-v8a"
            }
        }
    }
}
```
